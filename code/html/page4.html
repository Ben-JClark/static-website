When to use GET and POST?


Before we said that a GET request method is usually used to get read only data, this would
be done without modifying the server. We then said a POST request method is used when we
want to modify a server. We can also put these request methods into categories to better
understand which one we should use.

Safe methods

They are the methods that are intended for information retrieval only. They are GET, HEAD,
OPTIONS and TRACE.

Side effect methods

These methods have side effects on the server and can modify files and send emails.
These methods are POST, PUT, DELETE, PATCH.

Idempotent methods

Are methods that can be called multiple times with identical requests with only the first
successful one making an effect. These methods are PUT, DELETE, and all the safe methods.
